# 工业缺陷检测系统

## 项目简介

工业缺陷检测系统是一个集成了深度学习技术的智能工业缺陷检测平台，旨在通过计算机视觉技术自动识别和分析工业产品中的缺陷，提高检测效率和准确性。系统支持两种主要的缺陷检测方法：图像分割和目标检测，可以根据不同场景和需求灵活选择使用。

### 主要特点

- **双模型支持**：同时支持基于深度学习的图像分割和目标检测两种缺陷识别技术
- **自定义模型**：支持用户上传和加载自定义训练的模型文件
- **参数可调**：提供丰富的参数配置选项，允许用户根据需要调整检测精度和性能
- **批量处理**：支持单张和批量图像同时处理，提高工作效率
- **结果可视化**：直观展示检测结果，包括缺陷位置、类型和置信度
- **模型训练**：提供简易的模型训练接口，支持用户使用自有数据集进行模型训练

## 系统架构

系统采用前后端分离的架构设计，基于Flask框架构建，结合现代前端技术和深度学习模型，实现了高效的工业缺陷检测流程。

### 技术栈

- **后端**：Python 3.6+, Flask 2.0.1
- **深度学习框架**：TensorFlow 2.10.0, Ultralytics YOLO
- **图像处理**：OpenCV 4.8.0
- **前端**：HTML5, CSS3, JavaScript
- **数据存储**：文件系统（图像、模型文件存储）

## 功能模块

### 1. 图像分割模块

图像分割模块使用深度学习模型对工业产品图像进行像素级分割，精确识别缺陷区域。

#### 核心功能
- **缺陷区域检测**：精确识别图像中的缺陷区域
- **掩码生成**：为每个图像生成对应的缺陷掩码
- **结果可视化**：将检测到的缺陷区域叠加到原始图像上
- **自定义参数**：支持调整分割阈值、救援阈值、最小区域面积等参数

#### 技术实现
- 使用U-Net风格的深度学习模型
- 支持测试时数据增强(TTA)提高检测精度
- 包含图像预处理和后处理优化步骤
- 实现两阶段检测策略提高检测敏感度

### 2. 目标检测模块

目标检测模块基于YOLO模型，能够高效地检测图像中的缺陷位置，并提供边界框和置信度信息。

#### 核心功能
- **缺陷目标检测**：快速定位图像中的缺陷对象
- **边界框标注**：为每个检测到的缺陷生成边界框
- **置信度评分**：提供检测结果的置信度评分
- **批量处理**：高效处理大量图像

#### 技术实现
- 基于Ultralytics YOLO模型
- 支持多尺度检测提高小目标检测效果
- 提供可调的置信度阈值和IOU阈值
- 支持自定义模型加载

### 3. 模型训练模块

系统提供简易的模型训练接口，允许用户上传自己的数据集进行模型训练。

#### 核心功能
- **数据集上传**：支持ZIP格式的数据集上传
- **训练参数配置**：可配置训练轮次、批次大小、学习率等参数
- **模型保存**：自动保存训练完成的模型
- **训练结果展示**：提供训练指标如损失值、准确率等

#### 支持的训练类型
- **分割模型训练**：用于像素级缺陷分割
- **检测模型训练**：用于目标检测

### 4. 自定义模型加载

系统支持用户上传和加载自定义训练的模型，提供灵活的模型选择机制。

#### 支持的模型类型
- **分割模型**：支持.h5格式的TensorFlow模型
- **检测模型**：支持.pt格式的YOLO模型

#### 模型管理
- 自动保存用户上传的模型
- 提供模型加载状态反馈
- 支持在处理时选择不同模型

## 安装指南

### 环境要求

- Python 3.6或更高版本
- 推荐使用虚拟环境进行安装

### 安装步骤

1. **克隆项目代码**

   ```bash
   git clone <项目仓库地址>
   cd 工业检测
   ```

2. **创建并激活虚拟环境**

   ```bash
   # Windows系统
   python -m venv venv
   venv\Scripts\activate
   
   # Linux/MacOS系统
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **安装依赖**

   ```bash
   pip install -r requirements.txt
   ```

4. **下载预训练模型**（可选）

   将预训练的分割模型和检测模型分别放在`1/`和`2/models/`目录下，确保文件名正确。

5. **启动应用**

   ```bash
   python run.py
   ```

6. **访问系统**

   打开浏览器，访问 `http://localhost:5000` 进入系统。

## 使用说明

### 图像检测功能

1. **选择处理类型**
   - 在主界面选择「图像分割」或「目标检测」

2. **上传图像**
   - 点击「选择文件」按钮上传单张或多张图像
   - 支持常见图像格式如JPG、PNG等

3. **配置参数**（可选）
   - 根据需要调整相关参数：
     - **图像分割参数**：分割阈值、救援阈值、最小面积等
     - **目标检测参数**：置信度阈值、IOU阈值、图像尺寸等

4. **加载自定义模型**（可选）
   - 如需使用自定义模型，选择「自定义模型」选项
   - 上传相应格式的模型文件（分割为.h5，检测为.pt）
   - 点击「加载模型」按钮

5. **开始处理**
   - 点击「处理图像」按钮开始分析
   - 系统会显示处理状态

6. **查看结果**
   - 处理完成后，系统会显示原始图像和检测结果
   - 可以查看缺陷统计信息
   - 支持下载处理结果和对比图像

### 模型训练功能

1. **准备数据集**
   - 按照要求组织数据集结构：
     - **分割数据集**：包含images和masks两个文件夹
     - **检测数据集**：包含images和labels两个文件夹
   - 将数据集打包为ZIP格式

2. **选择训练类型**
   - 在训练页面选择「分割模型训练」或「检测模型训练」

3. **上传数据集**
   - 上传准备好的ZIP格式数据集

4. **配置训练参数**
   - 设置训练轮次(Epochs)
   - 设置批次大小(Batch Size)
   - 设置学习率(Learning Rate)
   - 设置图像尺寸(Image Size)

5. **开始训练**
   - 点击「开始训练」按钮
   - 系统会显示训练状态和进度

6. **查看训练结果**
   - 训练完成后，系统会显示训练指标
   - 训练好的模型会自动保存在系统中
   - 模型文件命名格式为 `trained_model_YYYYMMDD_HHMMSS.h5` 或相应的检测模型格式

## API文档

### 1. 模型加载API

#### 加载自定义模型

- **URL**: `/load_custom_model`
- **方法**: POST
- **参数**:
  - `model_file`: 文件，模型文件（分割模型为.h5格式，检测模型为.pt格式）
  - `model_type`: 字符串，模型类型（'segmentation'或'detection'）
- **返回**: JSON格式，包含成功状态、消息和模型信息

### 2. 图像上传API

#### 上传并处理图像

- **URL**: `/upload`
- **方法**: POST
- **参数**:
  - `files[]`: 文件，要处理的图像文件
  - `process_type`: 字符串，处理类型（'segmentation'或'detection'）
  - 其他相关参数根据处理类型而定
- **返回**: HTML页面，显示处理结果

### 3. 模型训练API

#### 训练模型

- **URL**: `/train_model`
- **方法**: POST
- **参数**:
  - `train_data`: 文件，ZIP格式的训练数据集
  - `training_type`: 字符串，训练类型（'segmentation'或'detection'）
  - 训练参数（如epochs, batch_size等）
- **返回**: JSON格式，包含训练状态、消息和训练结果

### 4. 文件下载API

#### 下载处理结果

- **URL**: `/download/result/<filename>`
- **方法**: GET
- **参数**: `filename`，要下载的结果文件名
- **返回**: 文件下载

#### 下载批量结果

- **URL**: `/download/batch`
- **方法**: GET
- **返回**: 包含所有结果的ZIP文件

#### 下载对比图像

- **URL**: `/download/comparison/<upload_folder>/<upload_filename>/<result_folder>/<result_filename>`
- **方法**: GET
- **参数**: 上传文件夹、上传文件名、结果文件夹、结果文件名
- **返回**: 对比图像下载

## 系统配置

### 配置文件

主要配置位于 `app/__init__.py` 文件中，包含以下关键配置项：

- **SECRET_KEY**: Flask应用密钥
- **STATIC_FOLDER**: 静态文件目录
- **UPLOAD_FOLDER**: 文件上传目录
- **SEGMENTATION_RESULT_FOLDER**: 分割结果保存目录
- **DETECTION_RESULT_FOLDER**: 检测结果保存目录
- **MAX_CONTENT_LENGTH**: 最大文件大小限制（默认16MB）

### 模型配置

模型加载配置位于 `app/model_loader.py` 文件中，包含：

- **SEGMENTATION_MODEL_PATH**: 默认分割模型路径
- **DETECTION_MODEL_PATH**: 默认检测模型路径
- **模型加载和验证逻辑**

### 检测参数配置

检测相关参数配置位于 `app/utils.py` 文件中，可根据需要调整：

- **分割参数**: 分割阈值、救援阈值、最小面积等
- **检测参数**: 置信度阈值、IOU阈值、图像尺寸等

## 故障排除

### 常见问题及解决方案

1. **模型加载失败**
   - 检查模型文件路径是否正确
   - 验证模型文件格式是否符合要求（分割为.h5，检测为.pt）
   - 查看控制台错误信息获取详细原因

2. **图像处理失败**
   - 确保上传的图像格式正确
   - 检查图像大小是否过大（超过16MB限制）
   - 尝试调整检测参数

3. **训练失败**
   - 验证数据集结构是否正确
   - 确保数据集ZIP文件包含必要的目录（images和labels/masks）
   - 检查训练参数设置是否合理

4. **系统启动失败**
   - 确保所有依赖已正确安装
   - 检查Python版本是否满足要求（3.6+）
   - 确认端口5000未被占用

## 性能优化建议

1. **提高检测速度**
   - 降低图像尺寸参数
   - 减少批量处理的图像数量
   - 调整检测/分割参数，降低处理精度要求

2. **提高检测精度**
   - 使用更高分辨率的输入图像
   - 调整分割阈值和救援阈值
   - 训练自定义模型时增加训练轮次
   - 使用更多样化的训练数据集

3. **系统资源优化**
   - 在生产环境中使用Gunicorn或uWSGI代替Flask开发服务器
   - 配置适当的内存和CPU资源
   - 考虑使用GPU加速模型推理
